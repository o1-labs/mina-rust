# Archive Nodes

o1Labs maintains archive node API infrastructure for both mainnet and devnet
networks. Archive nodes store complete blockchain history and provide database
query interfaces for historical data access.

## Available Archive Nodes

The following archive node APIs are officially maintained by o1Labs:

- **Devnet**:
  [`devnet-archive-node-api.gcp.o1test.net`](https://devnet-archive-node-api.gcp.o1test.net/)
- **Mainnet**:
  [`archive-node-api.gcp.o1test.net`](https://archive-node-api.gcp.o1test.net/)

<!-- prettier-ignore-start -->

:::info OCaml Implementation

These archive nodes run the OCaml implementation from the Mina Protocol
codebase. The mina-rust codebase does not currently have archive node
functionality, and o1Labs does not run any Rust archive nodes.

:::

<!-- prettier-ignore-stop -->

## Archive Node Capabilities

Archive nodes provide:

- **Historical blockchain data**: Complete transaction and block history
- **Database query interfaces**: SQL access to archived data
- **Transaction history**: Full account transaction records
- **Block and state archival**: Complete blockchain state snapshots

## Using Archive Nodes

### Database Connection

The archive node APIs provide HTTP access to query the PostgreSQL database.
These endpoints are primarily intended for querying historical blockchain data.
